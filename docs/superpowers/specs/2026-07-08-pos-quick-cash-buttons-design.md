# レジ画面：預かり金クイックボタン 設計仕様書

- 日付: 2026-07-08
- 対象: `pos/public/pos/` レジ画面
- ステータス: 設計承認済み・実装計画待ち

## 背景・目的

レジ画面の預かり金入力は現在、数値入力欄（キーボード直接入力）のみ。タッチ操作でPOSを使う場合、毎回オンスクリーンキーボードを呼び出して入力するのは手間がかかる。実際の会計でよく使う金額をワンタップで入力できるクイックボタンを追加し、タイプ（キーボード）とタッチの両方に対応させる。

## スコープ

含む:
- 預かり金入力欄の隣に「ぴったり」「500円」「1000円」の3ボタンを追加
- 各ボタンで預かり金欄を該当金額に**上書き**する
- カートが空（合計0円）のときはこの3ボタンを無効化する

含まない:
- 数値入力欄自体の変更（キーボード入力は現状のまま維持）
- 5000円・10000円ボタン（ユーザー判断により見送り）
- テンキー（0〜9のボタン）によるフル入力UI
- バックエンド（`pos/server/*.js`）の変更（既存の`cart:setReceived`イベントをそのまま再利用するため不要）

## 前提条件

- ホットドッグ200円・チーズホットドッグ300円という価格帯を踏まえ、500円・1000円が実用上よく使われる金額と判断
- 「ぴったり」はカート合計金額（`cart.total`）と同額を都度計算してセットする（固定値ではない）

## 設計

### UI
`pos/public/pos/index.html` の預かり金入力欄（`#received`）の隣に、以下3つのボタンを追加する:
```html
<button id="quickExact">ぴったり</button>
<button id="quick500">500円</button>
<button id="quick1000">1000円</button>
```

### 動作（`pos/public/pos/pos.js`）
- 各ボタンのクリックハンドラは、目的の金額を計算し `socket.emit('cart:setReceived', { amount })` を呼ぶ（既存の `#received` の `input` イベントハンドラと同じイベントを再利用する）
- 「ぴったり」は最新の `latestState.cart.total` を金額として使う
- 「500円」「1000円」は固定値
- `state` イベント受信時のレンダリング処理（`renderCart`）で、`cart.items.length === 0` のとき3ボタンとも `disabled` にする（既存の会計確定ボタンの無効化条件と同じ判定を流用）

### データフロー
```
ユーザーが「1000円」ボタンをタップ
  → pos.js: socket.emit('cart:setReceived', { amount: 1000 })
  → サーバー: 既存のcart:setReceivedハンドラがそのまま処理（変更不要）
  → 全クライアントへ state ブロードキャスト
  → pos.js: state受信 → #received の値が1000に更新される
```
既存の数値入力欄からの直接入力と全く同じサーバー処理経路を通るため、バックエンドの変更は不要。

### エラーハンドリング
既存の `OrderStore.setReceived()` のバリデーション（負数・非数値を拒否）がそのまま効くため、クイックボタン側で追加のバリデーションは不要（固定値・カート合計はいずれも正の数値であることが保証されている）。

### テスト
フロントエンドのみの変更のため、既存の他画面タスクと同様に自動テストは行わず、実ブラウザでの動作確認とする:
- カートが空の状態で3ボタンが無効化されていることを確認
- 商品追加後、各ボタンを押すと預かり金欄が正しい金額に変わることを確認
- 「ぴったり」がカート合計の変更（商品追加・削除）に追従することを確認
- ボタン操作後もキーボードでの手入力ができることを確認（上書きされるだけで、入力欄自体は無効化しない）
